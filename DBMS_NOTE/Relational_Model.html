üî∂ 1. What is the Relational Model?

Definition

The Relational Model is a way to store and manage data in tabular form. It was introduced by E.F. Codd in 1970.

  • Data is stored in Relations (Tables)
  • Each row = Tuple (Record)
  • Each column = Attribute (Field)
  • Each column has values from a Domain (Allowed values)

üî∂ 2. Keys (Uniqueness & Linking)

‚úÖ Primary Key

Uniquely identifies each row in a table.

‚ö†Ô∏è Cannot be NULL or duplicate.

Example: Roll_No in Student table

‚úÖ Candidate Key

All possible unique keys.

Example: Roll_No, Email

‚úÖ Alternate Key

Candidate keys that are NOT chosen as Primary Key.

Example: Email (when Roll_No is Primary)

‚úÖ Super Key

Any set of attributes that uniquely identifies rows.

‚úÖ Foreign Key

‚úÖ Composite Key

Combination of 2+ columns used as a Primary Key.

üîÑ Referential Integrity: INSERT, ON DELETE, and ON UPDATE

üéØ Purpose:

  • Maintain consistency between parent and child tables.
  • Handle what happens to child rows when a parent row is inserted, deleted, or updated.

üß© Example Tables:


CREATE TABLE Course (
    Course_ID VARCHAR(10) PRIMARY KEY
);

CREATE TABLE Student (
    Student_ID INT PRIMARY KEY,
    Course_ID VARCHAR(10),
    FOREIGN KEY (Course_ID) REFERENCES Course(Course_ID)
        ON DELETE CASCADE
        ON UPDATE CASCADE
);

‚ûï ON INSERT Rules

  • You cannot insert a foreign key in the child table if it does not exist in the parent table.
  • Problem: Trying to insert a Student with Course_ID = 'C999' when 'C999' is not in Course table.
  • Fix: First insert the corresponding row in the parent (Course) table: 
    
INSERT INTO Course (Course_ID) VALUES ('C999');
INSERT INTO Student (Student_ID, Course_ID) VALUES (101, 'C999');

üõ†Ô∏è ON DELETE Actions

  • ON DELETE CASCADE: Deletes child rows automatically
    Example: Deleting Course 'C101' also deletes all students enrolled in 'C101'
  • ON DELETE SET NULL: Sets foreign key to NULL in child rows
    Example: Deleting Course 'C101' sets Student's Course_ID to NULL
  • ON DELETE NO ACTION: Prevents deletion if child row exists
    Example: You can't delete Course 'C101' if any student is linked to it

‚úèÔ∏è ON UPDATE Actions

  • ON UPDATE CASCADE: Automatically updates child foreign key
    Example: Changing Course_ID from 'C101' to 'C202' also updates Student's Course_ID to 'C202'
  • ON UPDATE NO ACTION: Prevents update if foreign key is used in child
    Example: If 'C101' is used by students, you can't rename it

‚úÖ Summary Table:

Action Effect on Child
ON INSERT (invalid FK) Insert fails if referenced key is missing in parent
ON DELETE CASCADE Deletes matching child rows
ON DELETE SET NULL Sets foreign key to NULL
ON DELETE NO ACTION Prevents deletion if child exists
ON UPDATE CASCADE Updates foreign key in child
ON UPDATE NO ACTION Prevents update if child exists
