üß† 1. What is an Exception?

An exception is an event that disrupts normal flow of the program at runtime.

üî∫ Throwable Class

The Throwable class is the root of Java‚Äôs exception hierarchy. Every exception or error object in Java is a subclass of Throwable.

üì¶ Two Main Subclasses of Throwable

  • Exception ‚Äì for conditions that a program should catch and handle.
  • Error ‚Äì for serious problems that are not expected to be caught by applications.

üîç Exception Class (Recoverable)

The Exception class represents problems that can be handled programmatically.

‚úÖ Checked Exceptions

  • Checked at compile time.
  • Must be handled using try-catch or declared with throws.
  • Examples:
    • IOException
    • SQLException
    • ClassNotFoundException
    • FileNotFoundException
    • InterruptedException

‚ö†Ô∏è Unchecked Exceptions (Runtime)

  • Checked at runtime, not compile time.
  • Optional to handle explicitly.
  • Examples:
    • ArithmeticException
    • NullPointerException
    • IllegalArgumentException
    • NumberFormatException
    • IllegalStateException
    • UnsupportedOperationException
    • IndexOutOfBoundsException
      • ArrayIndexOutOfBoundsException
      • StringIndexOutOfBoundsException
    • ClassCastException

üö´ Error Class (Unrecoverable)

The Error class indicates serious problems that applications should not attempt to catch.

  • StackOverflowError ‚Äì occurs due to deep or infinite recursion.
  • OutOfMemoryError ‚Äì insufficient heap memory.
  • VirtualMachineError ‚Äì critical JVM failures.
  • AssertionError ‚Äì thrown when an assertion fails.
  • NoClassDefFoundError ‚Äì class not found at runtime.

üß© 3. try-catch-finally

üîß Syntax:

try {
    // Code that might throw an exception
} catch (ExceptionType e) {
    // Code to handle the specific exception
} finally {
    // Code that always executes (cleanup, closing files, etc.)
}

‚úÖ Example:

try {
    int result = 10 / 0;  // This will throw ArithmeticException
} catch (ArithmeticException e) {
    System.out.println("Cannot divide by zero!"); // This will run
} finally {
    System.out.println("Cleanup done!"); // This will always run
}

üìù Output:

  • Cannot divide by zero!
  • Cleanup done!

üìå Note: The finally block will always execute whether an exception is thrown or not. It's used to release resources like file handles, database connections, etc.

üß™ 4. throw vs throws

üß† Easy Explanation (Hinglish + English):

  • throw üëâ Is used when you want to actually throw an exception object.
  • throws üëâ Is used in method declaration to say "this method may throw an exception".
  • throw = action (exception ko fenkna)
  • throws = declaration (compiler ko batana ki exception aa sakta hai)
Keyword Meaning Used In Number of Exceptions
throw Used to manually throw a single exception object Inside method body / block Only one exception at a time
throws Used to declare exceptions that method may throw In method signature Multiple exceptions can be declared

‚úÖ Example 1: Using throw

void validateAge(int age) {
    if (age < 18) {
        // Throwing exception manually
        throw new IllegalArgumentException("Invalid age: must be 18 or above");
    }
    System.out.println("Valid age!");
}

public static void main(String[] args) {
    validateAge(15); 
}

Output: Invalid age: must be 18 or above

Note: Here we throw exception directly inside method body.

‚úÖ Example 2: Using throws

import java.io.*;

void readFile() throws IOException {
    // Declaring that this method may throw IOException
    FileReader fr = new FileReader("file.txt");
    System.out.println("File opened successfully!");
}

public static void main(String[] args) throws IOException {
    readFile(); 
}

Note: In method signature we wrote throws IOException ‚Üí this tells compiler that exception may come, so caller method (main) also needs to handle or declare it.

‚ö° Key Points to Remember

  • throw can throw only one exception at a time.
  • throws can declare multiple exceptions separated by commas.
  • throw is followed by an object (e.g., new Exception()).
  • throws is followed by a class name (e.g., IOException).
  • If you use throw, you must handle it (try-catch) or declare it (throws).

üí° Interview Tip:

Q: Difference between throw and throws?
A: throw is used to actually throw an exception object, while throws is used to declare that a method can throw exceptions.

üîÑ 6. final vs finally vs finalize()

üß† Easy Explanation (Hinglish + English):

  • final üëâ Use when you want to make something unchangeable (fix karna hai).
  • finally üëâ Block of code jo always execute hoga after try-catch (cleanup ke liye).
  • finalize() üëâ Method called by Garbage Collector before object destroy (‚ö†Ô∏è Deprecated in Java 9+).
Keyword Usage Where Used Important Note
final To prevent changes Variables, Methods, Classes Variable = constant, Method = cannot override, Class = cannot extend
finally Always executes for cleanup After try-catch Runs even if exception occurs or not
finalize() Cleanup before object destruction Method (called by Garbage Collector) Deprecated after Java 9 ‚Üí use try-with-resources

‚úÖ Example 1: Using final

    final void show() {
        System.out.println("Hello");
    }
}
class Child extends Parent {
    // void show() {} ‚ùå Error: cannot override final method
}

‚úÖ Example 2: Using finally

‚ùå Exception: / by zero
‚úÖ Finally block always runs
