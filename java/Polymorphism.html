üß† 1. What is Polymorphism?

Polymorphism means "many forms."

In Java, it allows one object to take many forms ‚Äî specifically, the same method name can behave differently depending on context.

üéØ Real-Life Analogy:

üõ† A remote control can operate a TV, a fan, or an AC ‚Äî same remote (method), but different actions depending on device (object).

‚ùì 2. Why Use Polymorphism?

  • Code Reusability
  • Flexibility to use parent reference for child objects
  • Helps in method overriding and interface implementation
  • Reduces coupling (object can be treated generally or specifically)

üîß 5. Method Overloading vs Method Overriding

‚úÖ Example of Method Overloading (Compile-Time):

class Calculator {
    int add(int a, int b) {
        return a + b;
    }

    double add(double a, double b) {
        return a + b;
    }
}

public class Main {
    public static void main(String[] args) {
        Calculator c = new Calculator();
        System.out.println(c.add(2, 3));        // calls int version
        System.out.println(c.add(2.5, 3.5));    // calls double version
    }
}

‚úîÔ∏è Same method name (add), different parameters = overloading

‚úÖ Example of Method Overriding (Runtime):

class Animal {
    void sound() {
        System.out.println("Animal makes sound");
    }
}

class Dog extends Animal {
    void sound() {
        System.out.println("Dog barks");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal a = new Dog(); // upcasting
        a.sound(); // Dog barks (overridden version) ‚Üí runtime polymorphism
    }
}

‚úîÔ∏è sound() method is called based on object type, not reference type.

üß≠ 6. Method Dispatch in Java

üîπ Dynamic Method Dispatch:

When a parent class reference is used to refer to a child class object, and overridden method is resolved at runtime, it's called dynamic method dispatch.

‚úÖ Example:

class Shape {
    void draw() {
        System.out.println("Drawing Shape");
    }
}

class Circle extends Shape {
    void draw() {
        System.out.println("Drawing Circle");
    }
}

public class Main {
    public static void main(String[] args) {
        Shape s = new Circle(); // upcasting
        s.draw(); // Output: Drawing Circle
    }
}

üìå The method call is resolved at runtime, depending on the actual object.

üîº 7. Upcasting & üîΩ Downcasting

üîº Upcasting (Safe, Common):

Assigning a child object to a parent reference.

Animal a = new Dog(); // Upcasting
a.sound(); // Allowed (calls overridden method)

‚úîÔ∏è Safe because the object is created as a Dog, but treated generally as an Animal.

üîΩ Downcasting (Risky):

Casting a parent reference back to a child object.

Animal a = new Dog(); // Upcasting
Dog d = (Dog) a;       // Downcasting
d.sound();             // Works if 'a' is actually a Dog
‚ö†Ô∏è Downcasting is risky because the compiler trusts you ‚Äî but if the object is not of the expected type, it throws a ClassCastException at runtime.

üí• ‚ùå Example that causes error:

Animal a = new Cat(); // Object is actually a Cat
Dog d = (Dog) a;       // ‚ö†Ô∏è Wrong downcasting
d.sound();             // ‚ùå Runtime Error: ClassCastException
üî• Output:
Exception in thread "main" java.lang.ClassCastException: Cat cannot be cast to Dog

‚úÖ Safer Way Using instanceof:

if (a instanceof Dog) {
    Dog d = (Dog) a;
    d.bark();
} else {
    System.out.println("Not a Dog. Cannot downcast safely.");
}
