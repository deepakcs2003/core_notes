‚úÖ What is a Process?

  • A process is a program in execution. OS handles processes.

‚úÖ How does OS create a process?

Steps:

  1. Load program & static data into memory.
  2. Allocate runtime stack (for function calls/variables).
  3. Allocate heap (for dynamic memory).
  4. Setup I/O tasks (open files, etc.).
  5. Give control to the main() function.

‚úÖ Process States (Life Cycle)

These are stages of a process:

State Description
NEW Process is being created
READY Loaded into RAM, waiting for CPU
RUNNING Actively executing on CPU
WAITING Waiting for I/O
TERMINATED Finished execution

üîÅ Some intermediate actions:

  • Preemption (from Running to Ready)
  • I/O request (from Running to Waiting)

‚úÖ Process Queues

  1. Job Queue: NEW processes in disk; managed by LTS (Long Term Scheduler).
  2. Ready Queue: READY processes in RAM; managed by STS (Short Term Scheduler).
  3. Waiting Queue: WAITING processes; doing I/O.

‚úÖ Schedulers

Scheduler Role Controls
LTS Loads new processes to RAM Degree of multiprogramming
STS Picks process from Ready Queue Assigns CPU
MTS Swaps out processes from RAM Medium-term memory use

‚úÖ Swapping (By MTS)

  • Temporarily swap out a process from RAM to disk to free memory.
  • Later, swap it back in when needed.
üìå Example: If RAM is full, MTS might suspend a low-priority process and bring it back later.

‚úÖ Context Switching

When CPU switches from one process to another:

  • Save current process state (in PCB)
  • Load new process state (from PCB)

Overhead: No actual work is done during switch.

üìå Analogy: Like saving your game before someone else plays.

‚úÖ Process Types

1. Orphan Process

  • Parent died, child still running.
  • Adopted by init process (PID 1).

2. Zombie Process

  • Child finished, parent hasn't read its status.
  • Stays in process table until parent calls wait().
üìå Example:
fork(); exit(0); // child wait(); // parent reaps

‚úÖ Preemptive vs Non-Preemptive Scheduling

Type Description
Non-Preemptive CPU is not taken forcefully; process runs till done or waits
Preemptive CPU can be taken anytime for a higher priority process
üìå Example: You're using PC ‚Üí Urgent system update interrupts ‚Üí You're paused (Preemptive)
