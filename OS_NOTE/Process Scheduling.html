1. Swapping

üéØ Interview Definition: Swapping is moving processes between main memory and secondary storage (disk) to manage memory efficiently.

üåç Real World Example: Your computer has 8GB RAM but you're running 20GB worth of programs. OS swaps inactive programs to disk to free up RAM for active ones. 
Swapping Process:
1. Process P1 is running in memory
2. P1 goes to waiting state (I/O operation)
3. Medium Term Scheduler swaps P1 to disk
4. Memory space freed for other processes
5. When P1 ready, swap back to memory
6. Continue execution from where it left off
Key Points:
  • Swap Out: Move process from memory to disk
  • Swap In: Move process from disk to memory
  • Purpose: Improve degree of multiprogramming
  • Who does it: Medium Term Scheduler (MTS)

2. Context Switching

üéØ Interview Key Point: Context switching is the process of saving current process state and loading another process state. It's pure overhead - no useful work is done during switching.

Context Switch Steps:
1. Save current process state in PCB
   - Program Counter (PC)
   - Register values
   - Memory management info
   - Process state
2. Select next process to run
3. Load new process state from PCB
4. Update memory management registers
5. Resume execution of new process
üåç Real World Example: Like a librarian helping customers - saves current customer's request details, helps next customer, then returns to first customer exactly where they left off.
Context Switch Overhead:
  • Time wasted in switching (no useful work)
  • Depends on hardware (registers, memory speed)
  • Typical time: 1-100 microseconds

3. Orphan and Zombie Processes

Aspect Orphan Process Zombie Process
Definition Parent terminated, child still running Child terminated, parent hasn't read exit status
State Running Terminated but entry in process table
Adoption Adopted by init process No adoption needed
Resource Usage Uses CPU, memory Only process table entry
üåç Real World Examples:
Orphan: Parent company shuts down, but subsidiary continues operating (gets new parent)
Zombie: Employee quits but HR hasn't processed their exit paperwork yet 
Orphan Process Example:
// Parent Process
int main() {
    pid_t pid = fork();
    if (pid == 0) {
        // Child process
        sleep(30);  // Child runs for 30 seconds
        printf("Child finished\n");
    } else {
        // Parent process
        printf("Parent exiting\n");
        exit(0);    // Parent dies immediately
    }
    // Child becomes orphan, adopted by init
}

Zombie Process Example:
// Parent Process
int main() {
    pid_t pid = fork();
    if (pid == 0) {
        // Child process
        printf("Child exiting\n");
        exit(0);    // Child dies immediately
    } else {
        // Parent process
        sleep(30);  // Parent sleeps for 30 seconds
        wait(NULL); // Finally reads child's exit status
    }
    // Child was zombie for 30 seconds
}
            

            

                üéØ Interview Questions:
                
Q: What happens to orphan processes?
                
A: They are adopted by init process (PID 1)
                
Q: How to prevent zombie processes?
                
A: Parent should call wait() to read child's exit status

4. Process Scheduling Introduction

üéØ Why Process Scheduling? To maximize CPU utilization by switching between processes when current process waits for I/O or time expires.

Process Scheduling Overview

Scheduling Type Preemptive Non-Preemptive
CPU Control OS can forcibly take CPU Process voluntarily gives up CPU
Starvation Less starvation More starvation possible
CPU Utilization High Low
Examples Round Robin, Priority FCFS, SJF
Important Formulas:
Turnaround Time (TAT) = Completion Time - Arrival Time
Waiting Time (WT) = Turnaround Time - Burst Time
Response Time = First CPU allocation time - Arrival Time

5. FCFS (First Come First Serve)

üéØ Interview Point: FCFS is simplest scheduling algorithm - processes are executed in order of arrival. Main problem is Convoy Effect.
Example: Processes P1(24), P2(3), P3(3) - numbers are burst times

Convoy Effect: Short processes wait behind long processes, like fast cars stuck behind slow truck on single-lane road.
üåç Real World: Bank queue - first customer takes long time, others wait regardless of their quick transactions.

7. Priority Scheduling

üéØ Interview Point: Each process has priority. Higher priority executes first. SJF is special case where priority = 1/burst_time.
Example: P1(10,3), P2(1,1), P3(2,4), P4(1,5) - (burst_time, priority)

Starvation Problem: Low priority processes may never execute.
Solution - Aging: Gradually increase priority of waiting processes.
Aging Implementation: Every 15 minutes: for each process in ready queue: if (waiting_time > 15 minutes): priority = priority + 1
üåç Real World: Airport boarding - first class boards first, but if economy passengers wait too long, they get priority (aging).

Conclusion

Process scheduling is a critical aspect of operating systems, ensuring efficient CPU utilization and responsiveness. Understanding various scheduling algorithms helps in designing better systems.
