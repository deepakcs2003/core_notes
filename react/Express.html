1. What’s Express.js in simple words?

Explanation: Express.js is like a helper for Node.js that makes building web apps or APIs super easy. It’s a framework that handles routing (directing users to the right page) and middleware (functions that process requests). It’s like a fast-food counter—quick, flexible, and gets the job done!

Code Example (if asked):

const express = require('express');
const app = express();
app.get('/', (req, res) => res.send('Hey, welcome!'));
app.listen(3000, () => console.log('Server’s up at port 3000!'));

How to Explain Code: Require Express, create an app instance, add a GET route, and listen on a port. It’s async and non-blocking for handling many users.

Pro Tip: Mention Express is lightweight and flexible, unlike raw Node.js, which is like cooking from scratch—Express saves time!

2. How do you install Express.js?

Explanation: You add Express to your project using npm, like downloading an app. It’s stored in your project’s folder for consistency.

Code Example (if asked):

npm init -y  # Sets up package.json
npm install express  # Adds Express

How to Explain Code: Init creates a package file, then install adds Express.

Pro Tip: Use a specific version (e.g., `npm install express@4.18.2`) for production stability, like locking a recipe.

3. How do you set up a basic Express server?

Explanation: It’s like starting a food truck. Create an Express app, define routes, and listen on a port. Here’s a diagram of the HTTP request-response cycle:

Code Example (if asked):

const express = require('express');
const app = express();
app.get('/', (req, res) => res.send('Welcome to my food truck!'));
app.listen(3000, () => console.log('Truck’s open at http://localhost:3000'));

How to Explain Code: Require Express, create app, add route, listen. Reference the diagram: Client sends request, server processes, sends response.

Pro Tip: `app.listen` uses Node’s HTTP server, great for handling lots of requests.

4. What’s routing in Express?

Explanation: Routing is like a GPS for your app—it directs users to the right page based on URL and method (GET, POST). Here’s a routing diagram:

Code Example (if asked):

app.get('/home', (req, res) => res.send('You’re home!'));
app.post('/order', (req, res) => res.send('Order placed!'));

How to Explain Code: Use app.get for reads, app.post for creates. Diagram shows routes directing to controllers.

Pro Tip: Use patterns like `/user/:id` for dynamic URLs. Order specific routes before general ones.

5. What’s middleware in Express?

Explanation: Middleware is like a chef checking ingredients before cooking. It runs during a request (e.g., logging, auth). Here’s a middleware chain diagram:

Code Example (if asked):

app.use((req, res, next) => {
  console.log('Got a request for:', req.url);
  next();
});

How to Explain Code: Logs URL, calls next(). Diagram shows request through middleware to response.

Pro Tip: Middleware is Express’s backbone—split tasks like logging. Avoid heavy middleware early to prevent slowdowns.

6. What does `next()` do in middleware?

Explanation: `next()` passes the request to the next middleware or route, like passing a plate to the next chef.

Code Example (if asked):

app.use((req, res, next) => {
  if (req.query.vip === 'true') next();
  else res.send('Sorry, VIPs only!');
});

How to Explain Code: Checks VIP query, calls next() or sends response.

Pro Tip: `next('error')` jumps to error middleware. Use after `await` in async code.

7. How do you serve static files (like images or CSS)?

Explanation: Static files are like ready-to-serve snacks. Use `express.static` to serve files from a folder.

Code Example (if asked):

app.use(express.static('public'));

How to Explain Code: Serves files in 'public' directly (e.g., /style.css).

Pro Tip: Use a “public” folder for safety. Pair with a CDN for speed.

8. What’s `req.params`?

Explanation: Grabs dynamic URL parts, like `/user/123` where `123` is in `req.params.id`.

Code Example (if asked):

app.get('/user/:id', (req, res) => res.send(`User ID: ${req.params.id}`));

How to Explain Code: :id captures the URL value.

Pro Tip: Validate `req.params` to prevent hacks.

9. How do you send JSON in Express?

Explanation: Use `res.json()` to send data as JSON, perfect for APIs.

Code Example (if asked):

app.get('/api/menu', (req, res) => res.json({ dish: 'Pizza', price: 10 }));

How to Explain Code: Sends object as JSON with correct headers.

Pro Tip: `res.json()` handles edge cases better than `res.send()`. Use status codes for REST APIs.

10. What HTTP methods does Express support?

Explanation: Supports all: GET, POST, PUT, DELETE, PATCH, etc. Like different customer requests.

Pro Tip: Use for REST APIs (e.g., GET `/users`). Handle CORS for cross-origin requests.

11. How do you handle a 404 (page not found)?

Explanation: Add a catch-all middleware to handle unmatched routes.

Code Example (if asked):

How to Explain Code: Runs last, sets 404 status.

Pro Tip: Return JSON for APIs or log 404s for analytics.

12. What’s `express.Router()`?

Code Example (if asked):

How to Explain Code: Create router, add routes, mount to app.

13. How do you add middleware to a specific route?

Explanation: Add middleware before the route handler.

Code Example (if asked):

app.get('/secret', checkUser, (req, res) => res.send('Secret menu!'));

How to Explain Code: Middleware runs before handler.

Pro Tip: Great for route-specific auth. Chain multiple middleware.

14. What are the arguments of middleware functions?

Explanation: Middleware gets `req`, `res`, `next`. Error middleware gets `err` too.

Pro Tip: `req` and `res` are mutable, like adding notes to an order.

15. What’s the difference between `app.use()` and `app.get()`?

Explanation: `app.use()` runs middleware for all requests; `app.get()` handles only GET requests.

Pro Tip: Use `app.use()` for global tasks, `app.get()` for specific pages.

23. Why separate the Express app and server?

Explanation: Keep app (routes, middleware) and server (`app.listen`) separate.

Code Example (if asked): (app.js)

app.listen(3000);

How to Explain Code: Export app, import and listen.

Pro Tip: Helps testing and scaling.

36. What’s the difference between `req.params`, `req.query`, and `req.body`?

Explanation:

  • `req.params`: From URL paths (e.g., `/user/123` → `req.params.id`).
  • `req.query`: From queries (e.g., `?name=John` → `req.query.name`).
  • `req.body`: From POST/PUT data.

Pro Tip: Use `params` for IDs, `query` for filters, `body` for payloads. Secure `body` with HTTPS.

MongoDB Integration with Express.js

Added 5 questions on MongoDB for deeper understanding.

41. How do you connect MongoDB to Express.js?

Explanation: Use `mongoose` to connect to MongoDB. Here’s a connection flow diagram:

Code Example (if asked):

const mongoose = require('mongoose');
mongoose.connect('mongodb://localhost:27017/mydb', { useNewUrlParser: true })
  .then(() => console.log('Connected to MongoDB'))
  .catch(err => console.error('Connection error', err));

How to Explain Code: Connect with DB URL, handle success/error. Diagram shows Client → Express → MongoDB.

Pro Tip: Use environment variables for DB URL. Handle reconnection in production.

42. What is Mongoose?

Explanation: Mongoose is an ODM for MongoDB, providing schema-based data modeling.

Pro Tip: Translates JS objects to MongoDB documents, simplifies NoSQL.

43. How to define a schema in Mongoose?

Explanation: Schemas define document structure. Here’s a schema diagram:

Code Example (if asked):

const mongoose = require('mongoose');
const userSchema = new mongoose.Schema({
  name: { type: String, required: true },
  age: Number
});
const User = mongoose.model('User', userSchema);

How to Explain Code: Define schema, compile model. Diagram shows Node.js → Mongoose → MongoDB.

Pro Tip: Add validators for data integrity.

44. How to perform CRUD operations with Mongoose in Express?

Explanation: Use model methods for Create, Read, Update, Delete.

Code Example (if asked): (Create)

app.post('/users', async (req, res) => {
  const user = new User(req.body);
  await user.save();
  res.send(user);
});

(Read): await User.find();

(Update): await User.updateOne({ _id: id }, { $set: { name: 'New' } });

(Delete): await User.deleteOne({ _id: id });

How to Explain Code: Use async/await for DB operations.

Pro Tip: Use try-catch, populate() for relationships.

45. How to handle MongoDB errors in Express?

Explanation: Catch MongoDB errors with Express error middleware.

Code Example (if asked):

app.post('/users', async (req, res, next) => {
  try {
    const user = new User(req.body);
    await user.save();
    res.send(user);
  } catch (err) {
    next(err);
  }
});

How to Explain Code: Try-catch passes errors to next().

Pro Tip: Log specific errors (e.g., DuplicateKey) with Winston.
